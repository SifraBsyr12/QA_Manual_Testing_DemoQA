# QA Manual Testing - DemoQA

## Project Overview
This repository contains **manual test cases and bug reports** for the DemoQA website.  
The purpose of this project is to demonstrate **QA testing skills**, including:  

- Functional testing  
- Validation testing  
- Security testing (SQL/JS injection)  
- UI/UX validation  

The test cases cover **Login and Forms** modules of DemoQA.

## How to Use This Repository

1. **View Test Cases:**  
   - Open the `Test_Cases` folder to see individual `.md` files for Login and Forms.  
   - Each file includes: Steps, Expected Result, Actual Result, Status, and Screenshot path.

2. **View Bug Reports:**  
   - Open the `Bug_Reports` folder to see detailed bug reports.  
   - Each bug report links to a related test case (via `Related Test Case`).

3. **Screenshots:**  
   - Screenshots are stored in the `Screenshots` folder.  
   - They provide evidence for Pass/Fail results of test cases.

---

## How to Execute Tests (Manual)

1. Open the target page of DemoQA:
   - Login Page: [https://demoqa.com/login](https://demoqa.com/login)  

   - Forms Page: [https://demoqa.com/automation-practice-form](https://demoqa.com/automation-practice-form)  

  

2. Follow the **Steps to Execute** in each test case file.  

3. Observe the outcome and fill **Actual Result** and **Status (Pass/Fail)** in the test case file.  

4. If the test fails, create a **Bug Report** in the `Bug_Reports` folder and attach relevant screenshots.
